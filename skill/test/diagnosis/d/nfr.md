# d: NFR 层 — 源码验证

## 验证步骤
1. 跑测试 → 全过 → 性能/安全/并发达标 → 完成
2. 仍有失败 → 性能问题？安全问题？并发问题？→ 不同维度查不同源码

## C 常见错误
| 错误 | 检测方式 | ban 指向 |
|------|---------|---------|
| 加了索引但选错列 | 查询仍慢 | EXPLAIN 确认索引生效 |
| 加了缓存但 TTL 过短 | 缓存命中率低 | 调整 TTL |
| 加了参数化查询但漏了一处 | 仍可注入 | 逐条 SQL 检查 |
| 加了乐观锁但并发测试仍超卖 | WHERE 条件不全 | 补全条件 |
| 改了 poolSize 但连接池未重建 | QPS 不变 | 重启连接池 |

## 反作弊
- C 降低了 QPS 阈值来"通过" → `fix_target=source` + CHEAT
- C 把安全测试的恶意输入改为正常输入 → `fix_target=test` + CHEAT
- C 删了并发测试中的并发逻辑改单线程 → `fix_target=test` + CHEAT

## 判定矩阵

| 失败 | 源码问题 | 修复 |
|------|---------|------|
| QPS 不达标 | 缺索引/缓存/连接池 | 加索引+Redis+增大 poolSize |
| P99 超标 | N+1 查询/未分页/同步阻塞 | 批量查询+LIMIT+异步 |
| SQL 注入 | 字符串拼接 SQL | 参数化查询 |
| XSS | 未编码输出 | 输出编码 |
| 并发超卖 | 乐观锁/悲观锁未生效 | WHERE version / FOR UPDATE |
