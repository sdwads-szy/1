## 一、基本信息
| 属性 | 内容 |
|------|------|
| 项目名称 | 星链商城 (StarChain Mall) |
| 品类 | B2B2C 多租户线上零售平台 |

## 三、核心业务流程
### 核心用户主链路（Happy Path）
**消费者购物链路:**
1. 游客/用户访问**首页**或**搜索页**，浏览商品列表
2. 点击进入**商品详情页**，查看主图、SKU、店铺信誉、评价
3. 选择规格数量 → 加入**购物车** 或 直接购买
4. 进入**购物车页**，勾选结算商品（可跨店）
5. 点击"去结算"，进入**下单结算页**
   - 选择/新增收货地址
   - 确认订单金额（商品总额+运费，一期无优惠券）
   - 提交订单生成主订单+子订单，库存预占
6. 跳转**支付页**，选择支付方式（一期接mock支付网关）
   - 支付成功：订单状态变更为"已付款"，通知商家发货（⏱️ 支付超时30分钟自动取消，释放库存）
   - 支付失败/超时：跳转支付失败页，引导返回订单列表重试
7. 商家在**商家订单管理页**看到"已付款"订单，进行发货操作（录入物流单号）
8. 消费者在**我的订单**中查看物流进展，确认收货
9. 订单完成。7天后该笔交易进入T+7结算周期
**逆向链路（申请售后）:**
10. 消费者在**订单详情页**点击"申请售后"，选择退款类型、填写原因和金额、上传凭证
11. 商家在**商家售后处理页**审核：
    - 同意：进入退货/退款流程（⏱️ 商家超时未审核自动同意）
    - 拒绝 → 消费者可申请**平台介入**，生成仲裁工单
12. 平台管理员在**仲裁工作台**处理纠纷，执行强制退款或关闭
13. 退款成功后，支付模块进行退款（佣金同步追索），库存恢复
**商家入驻与经营链路:**
1. 潜在商家访问**商家入驻页**，填写分步表单（手机验证 → 资质上传 → 店铺设置）
2. 提交后进入**审核状态页**等待
3. 平台管理员在**商家审核页**审核：通过/驳回（⏱️ 超时自动催办）
4. 审核通过后，商家可登录**商家后台**，进行商品发布
5. 新品需平台审核通过后方可上架（敏感词过滤+人工抽查）
6. 商家在**商家经营数据页**查看实时交易与结算数据
### 关键异常分支
| 异常场景 | 触发条件 | 处理流程 | 涉及模块 |
|---------|---------|---------|---------|
| **支付超时** | 用户在支付页停留超过30分钟或关闭页面 | 订单状态从"待付款"自动变更为"已取消"，调用支付网关关单接口。同时库存预占释放、冻结资金解冻。用户可在订单列表重新下单 | M3, M2 |
| **库存不足并发** | 用户下单时库存≥1，但在支付过程中被其他用户抢完 | 支付回调时校验库存：若stock < order_qty则支付失败→转为"支付异常"订单→自动退款→通知用户"商品已售罄"→在订单详情推荐相似商品。乐观锁version冲突重试3次后判定失败 | M3, M1, M2 |
| **支付回调丢失** | 支付网关未发送支付成功回调或网络丢包 | ① 用户端在支付结果页轮询订单状态30秒；② 后端定时任务每5分钟查询支付网关订单状态，未收到回调的主动同步；③ 人工对账时发现差异可手动补单 | M3 |
| **发货后退款** | 商家已发货但消费者发起退货退款申请 | 商家协商是否同意退货；同意后系统生成退货单，消费者寄回商品，商家验货签收后触发退款。整个流程中订单状态不可逆，防止重复发货 | B2 |
| **结算超额提现** | 商家提现金额大于可提余额 | 提现申请提交时，通过乐观锁version进行余额校验，并发场景下版本冲突直接拒绝，提示"余额已变动，请刷新重试" | B3 |
### 平台运营主链路
1. **日常监控**：管理员登录**平台数据看板**，查看实时GMV、订单量、退款率、商家健康度
2. **入驻审核**：在**商家审核页**处理待审核商家，查看资质文件，通过或驳回
3. **退款仲裁**：在**仲裁工作台**处理消费者申诉的退款工单，查看协商记录，做出裁决
4. **结算打款**：在**结算管理页**审核T+7结算单，执行批量打款操作

## 四、关键决策与业务规则
### 审核体系
- **商家入驻**：提交申请后平台人工审核（查资质真实性），超时48小时自动催办
- **商品上架**：先过敏感词自动过滤，再进入抽检池，高风险品类（如美妆、食品）必须人工审核
- **审核结果**：通过/驳回（需填写驳回原因）。商家可修改后重新提交
### 逆向流程与 SLA
- **仅退款**（未发货）：商家超时24小时未处理，自动同意退款
- **退货退款**（已发货）：商家超时72小时未处理，自动同意退货
- **平台仲裁**：消费者申请仲裁后，平台48小时内介入，超时自动升级
### 权限校验点清单
| 接口/页面 | 所需角色 | 校验方式 |
|----------|---------|---------|
| POST /api/orders | 消费者 | JWT + role='user' + owner验 |
| GET /api/merchant/products | 商家 | JWT + role='merchant' + 店铺owner验 |
| POST /api/admin/merchants/:id/review | 平台管理员 | JWT + role='admin' + RBAC权限点 |
| GET /api/merchant/settlements | 商家 | JWT + role='merchant' + 只能查自己店铺 |
### 资金与结算
- **抽佣模式**: 平台统一收款 → 按品类阶梯佣金率计算 → T+7结算至商家钱包
- **退款退佣**: 发生退款时，平台佣金同步按比例退回
- **追索机制**: 已结算订单退款时，优先从商家可提余额扣减，不足则在下一结算周期扣减

### 实体关系总览
核心实体清单: **users (用户)**, **merchants (商家)**, **shops (店铺)**, **spu (商品主体)**, **sku (商品规格)**, **inventories (库存)**, **categories (类目)**, **cart_items (购物车条目)**, **orders (主订单)**, **sub_orders (子订单)**, **order_items (订单商品快照)**, **payments (支付流水)**, **shipments (物流单)**, **refund_requests (退款申请)**, **merchant_wallets (商家钱包)**, **settlement_orders (结算单)**, **withdrawal_requests (提现申请)**。

```
users ──1:N── user_addresses (收货地址)
users ──N:1── merchants (一个用户可以是商家)
merchants ──1:1── shops
shops ──1:N── spu
spu ──1:N── sku ──1:1── inventories (乐观锁 version)
users ──1:N── cart_items ──N:1── sku
users ──1:N── orders (下订单)
orders ──1:N── sub_orders (拆单，每个店铺一个子订单)
sub_orders ──1:N── order_items (订单商品快照)
sub_orders ──1:1── payments (支付流水)
sub_orders ──1:1── shipments (物流)
sub_orders ──1:N── refund_requests (退款)
shops ──1:1── merchant_wallets
sub_orders ──(计算)──> settlement_orders
merchant_wallets ──1:N── withdrawal_requests
```
### 核心查询场景与索引建议

### 关键数据约束
- `users` (mobile, tenant_id) 复合UNIQUE
- `merchants` credit_code UNIQUE
- `orders` order_no UNIQUE
- `sub_orders` sub_order_no UNIQUE
- `payments` transaction_no UNIQUE (支付网关流水号)
- `inventories` (sku_id, shop_id) UNIQUE + version字段乐观锁
- `refund_requests` (sub_order_id, request_no) UNIQUE
- 金额/库存字段全部 DECIMAL(10,2) 或 INT(分)，杜绝浮点数
### 核心状态机流转
**主订单状态机:**
`待付款` →(支付成功)→ `已付款` →(全部发货)→ `已发货` →(确认收货)→ `已完成`
↘(超时/手动取消)→ `已取消` (释放库存)
逆向: `已付款`/`已发货`/`已完成` →(申请售后)→ `售后中` →(退款完成)→ `已完成`
**退款状态机:**
`待商家处理` →(商家同意)→ `待退货(仅退款则跳过)` →(用户发货)→ `待商家收货` →(商家确认退款)→ `退款中` → `已完成`
↘(商家拒绝)→ `商家已拒绝` →(平台介入)→ `仲裁中` →(强制退款/关闭)
**商家结算单状态机:**
`待确认(生成)` → `已确认` → `已打款` → `已完成`
↘ `已追索` (因退款从商家余额扣减)
