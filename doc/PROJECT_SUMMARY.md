# 项目总结

## 项目背景

针对大语言模型生成项目时易出现跨模块接口不一致、上下文遗忘及幻觉等问题，设计并实现了一套契约驱动的多智能体代码生成系统。系统根据一句话需求自动完成需求分析、架构设计、代码生成、测试验证及自动修复，产出包含前端、后端及数据库的完整可运行全栈项目。

## 核心工作

### 1. 需求分析阶段

解决从"一句话需求"到"完整需求文档"的跨越。

单一 Agent 过长的思考链会导致角色混淆，因此采用多 Agent 系统——1 个 PM 主理人每轮固定一个话题，配合 10 个不同身份 Agent（客户、用户、运营、创新、行业专家、质量、数据、架构、安全、设计）进行多角度讨论，PM 负责冲突裁决防止角色间矛盾堆积。为防止上下文过长，使用 Memory 保留每轮讨论结果和摘要，下一轮仅使用摘要了解已确定内容，保证多轮讨论不丢失信息的同时控制 token 消耗。

### 2. 架构设计阶段

解决从"需求文档"到"架构设计"的衔接。

需求文档直接到架构设计过于跳跃，因此设计场景树作为中间产物，将需求抽象为 7 种契约关系——场景、业务逻辑、API 接口、DB-API 接口、页面导航、认证、数据模型——场景树覆盖 PRD 全部模块和实体，后续作为唯一基准校验契约完整性。为解决数据一致性和幻觉问题，采用"从上往下规划契约（plan_tree + §7 计数校验强制补全），从下往上分配任务（data→interfaces→business→tasks，每层只引用前序已生成的契约）"的策略，分层构建契约和任务，每层引用前序产出保持一致性。

### 3. 源代码生成

解决从"契约"到"代码"的实现。

使用依赖拓扑排序控制任务执行顺序，按类型分层（infra→db→frontend→backend→integration），层间串行保证依赖、层内并发最大化效率。通过剪枝算法防止上游错误级联传播——上游失败则依赖它的下游任务被剪枝，避免在不可靠基础上继续生成。设计真理传递机制——下游 Agent 通过加载上游产出的真理文件（接口签名/字段名/表结构的结构化 JSON）了解已有代码，而非依赖 LLM 记忆，从根本上消除跨模块接口不一致。

### 4. 测试任务生成

解决从"架构任务"到"测试设计"的衔接。

与架构设计类似，先构建测试场景树（从上往下全量规划），再分层生成 5 大类型（静态层/接口层/逻辑层/场景层/质量层）、22 个子层的 173 个测试任务。静态层使用 checkPoints（确定性检查点，如"导出 authenticate 函数"），其他层使用 testScenarios（场景化验证集，覆盖正向/异常/边界/鉴权四个维度）。每个测试任务通过 dependencies 显式声明对上游测试任务的依赖，保证测试执行的有序性。

### 5. 测试与修复

解决源代码错误的自动发现与修复，是整个系统的收敛验证闭环。

采用依赖拓扑排序控制执行顺序，一层未全过后后续全部阻塞，保证测试的可靠性。静态层（infra/db/frontend/peer_deps/integ）使用 Python 确定性校验——语法、导入导出关系、DDL vs 契约对比，直接完成不进入循环。在此基础上执行 ABCD 四步循环：A 测试编写（基于模板+真理+场景生成测试代码）→ B 测试诊断（跑测试后区分测试代码问题与源码问题，输出 Ban 指纹）→ C 源码修复（按 Ban 精确修复，严格限制读写文件次数）→ D 验证修复（重跑测试，通过输出真理/失败输出新 Ban）。每步严格限制工作内容和工具调用次数，配合 (TS,SS)/(TF,SS)/(TS,SF) 三种状态跳过已完成步骤减少 token 消耗。采用 Ban 记忆 + 真理传递的双记忆机制，保证跨 Agent 跨轮次信息不丢失，多轮迭代收敛至全部通过。

### 6. 知识库构建

解决过去问题重复踩坑与代码质量的持续提升。

设计图锚点 + 语义向量的双索引知识库结构。经验来源于 ABCD 循环的 Ban 记忆，经过 Python 规则提取泛化标签（120+ 关键词映射）、LLM 净化（去噪+结构化）、Python 精确去重 + 语义去重（相似度 ≥ 0.95 跳过、0.85~0.95 LLM 合并）、向量化后存入 LanceDB，配合 IVF_PQ 索引。检索采用合约锚点→标签→泛化模式标签三级回退，确保跨项目也能命中相关经验。通过评分机制（测试通过率 50% + 代码规范 30% + 记忆修复率 20%）将 S/A 级代码函数级入库，作为后续生成的参考模板，实现代码质量的持续提升。

| 控制器 | 23 个 | 含错误码映射的统一响应模式 |

| 中间件 | 3 个 | JWT 认证、权限校验、错误处理 |
| SQL 表 | 25 张 + 26 种子数据 | DDL 与契约逐字段一致 |
| Vue 页面 | 31 个 | 含路由懒加载、角色守卫、Token 过期处理 |
| API 模块 | 24 个 | 前端 API 调用，端点与后端路由对齐 |
| 测试文件 | 57 个 | 含 94 断言的 t_dbapi_order |

验证了契约驱动多智能体协作在自动化全栈代码生成中的可行性。
